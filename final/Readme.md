# M-FIG and Block-QAOA based Hybrid Deepfake Detection

## 1. Basic Description

This repository contains the source code and sample dataset for the final graduation project of Team Qubit.

The project implements a hybrid deepfake detection pipeline that combines classical deep learning-based feature extraction with quantum optimization-based feature selection. The final goal is to classify face images into two categories:

```text
REAL
FAKE
```

The main executable source code is provided in:

```bash
final.py
```

The dataset and metadata file are included under:

```bash
archive/dataset_processed_split/
```

---

## 2. Project Description

Deepfake detection is an important task for identifying manipulated facial images generated by deep learning-based image synthesis or face manipulation techniques.

In this project, we propose a hybrid pipeline that consists of the following major components:

1. Face detection and preprocessing using MTCNN
2. Feature extraction using pretrained ResNet18
3. Multi-Feature Interaction Graph, or M-FIG, construction
4. Feature block clustering using agglomerative clustering
5. Block-wise feature selection using QAOA
6. Final binary classification using a quantum-inspired attention classifier

The proposed method first extracts 512-dimensional image features from face images. Then, the feature space is divided into multiple blocks based on feature interaction patterns. Within each block, QAOA is used to select important and less redundant features. Finally, the selected features are used to train a binary classifier.

The program prints the final evaluation metrics after training and testing.

## 3. Source Code Description

The main source code is contained in:

```bash
final.py
```

The source code is organized into the following parts.

### 3.1 Environment and Hyperparameter Settings

The code defines the dataset path and main hyperparameters.

```python
BASE_DIR = os.path.join("archive", "dataset_processed_split")
CSV_PATH = os.path.join(BASE_DIR, "dataset_manifest.csv")

NUM_BLOCKS_K = 32
TARGET_FEATURES_TOTAL = 256
LAPLACIAN_THRESHOLD = 100.0
```

The program automatically uses GPU if CUDA is available.

```python
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
```

---

### 3.2 Dataset Class

The `DeepfakeDataset` class loads image paths and labels from `dataset_manifest.csv`.

It performs the following operations:

1. Loads the metadata CSV file
2. Selects data according to the split value, such as `train` or `test`
3. Reads each image using OpenCV
4. Converts the image from BGR to RGB
5. Removes blurry training images using Laplacian variance
6. Detects and crops the face region using MTCNN
7. Returns the processed face tensor and label

The label is converted as follows:

```text
REAL → 0
FAKE → 1
```

---

### 3.3 ResNet18 Feature Extractor

The function `get_backbone()` loads a pretrained ResNet18 model.

The final classification layer is removed, so the model is used only as a feature extractor. Each input face image is converted into a 512-dimensional feature vector.

---

### 3.4 Feature Extraction

The function `extract_features()` receives a dataloader and a backbone model.

It extracts feature vectors from all valid images and returns:

```text
X: extracted feature matrix
y: label vector
```

Images with invalid labels or failed preprocessing are skipped.

---

### 3.5 M-FIG Construction and Feature Clustering

The function `build_mfig_and_cluster()` constructs the feature interaction structure.

It calculates:

1. Mutual information scores between features and labels
2. Euclidean distances between feature activation patterns
3. Absolute Pearson correlation coefficients between features

Then, agglomerative clustering is used to divide the original 512-dimensional feature space into several feature blocks.

---

### 3.6 Block-QAOA Feature Selection

The function `select_features_qaoa()` performs feature selection block by block.

For each feature block, the code formulates a QUBO problem. The QUBO objective is designed to:

1. Select features with high mutual information
2. Penalize redundant features with high correlation
3. Control the number of selected features

QAOA is used to solve the optimization problem.

If QAOA fails for a block, the code automatically uses a greedy mutual-information-based fallback method.

---

### 3.7 Quantum-inspired Attention Classifier

The `QuantumHybridClassifier` class is the final classifier.

It consists of:

1. Born's rule-inspired trainable probability weights
2. Multi-head attention layer
3. Fully connected binary classification layers

The classifier outputs a probability value between 0 and 1.

```text
Probability close to 0 → REAL
Probability close to 1 → FAKE
```

---

### 3.8 Main Pipeline

The `main()` function executes the full experiment pipeline.

The execution order is:

```text
Load dataset
→ Extract ResNet18 features
→ Build M-FIG
→ Cluster feature blocks
→ Select features with Block-QAOA
→ Train quantum-inspired classifier
→ Evaluate on test data
→ Print final metrics
```

The final output includes:

```text
Accuracy
Precision
Recall
F1-Score
AUC-ROC
```

## 8. Database or Data Used

This project uses an image dataset for deepfake detection.

The dataset is organized by split and fake type. The expected structure is:

```bash
archive/dataset_processed_split/
├── train/
│   ├── Real/
│   ├── DeepFakeDetection/
│   ├── Deepfakes/
│   ├── Face2Face/
│   ├── FaceShifter/
│   ├── FaceSwap/
│   └── NeuralTextures/
│
├── val/
│   ├── Real/
│   ├── DeepFakeDetection/
│   ├── Deepfakes/
│   ├── Face2Face/
│   ├── FaceShifter/
│   ├── FaceSwap/
│   └── NeuralTextures/
│
└── test/
    ├── Real/
    ├── DeepFakeDetection/
    ├── Deepfakes/
    ├── Face2Face/
    ├── FaceShifter/
    ├── FaceSwap/
    └── NeuralTextures/
```

The metadata file is:

```bash
archive/dataset_processed_split/dataset_manifest.csv
```

The CSV file should contain at least the following columns:

| Column      | Description                                                    |
| ----------- | -------------------------------------------------------------- |
| `split`     | Dataset split, such as `train`, `val`, or `test`               |
| `label`     | Ground-truth label, either `REAL` or `FAKE`                    |
| `fake_type` | Subdirectory name, such as `Real`, `Deepfakes`, or `Face2Face` |
| `filename`  | Image file name                                                |

The image path is constructed as:

```bash
archive/dataset_processed_split/{split}/{fake_type}/{filename}
```

For example:

```bash
archive/dataset_processed_split/train/Real/example.jpg
archive/dataset_processed_split/test/Deepfakes/example.jpg
```

## 10. Troubleshooting

### 10.1 Dataset Path Error

If the dataset file cannot be found, check whether the following file exists:

```bash
archive/dataset_processed_split/dataset_manifest.csv
```

Also make sure that the code is executed from the `final` directory:

```bash
cd Qubit_research/final
python final.py
```

---

### 10.2 Missing Train or Test Data

The current code requires both `train` and `test` splits.

Check whether the following folders exist:

```bash
archive/dataset_processed_split/train/
archive/dataset_processed_split/test/
```

Also check whether `dataset_manifest.csv` contains rows whose `split` column is `train` and `test`.

---

### 10.3 ModuleNotFoundError

If a module is missing, run:

```bash
pip install -r requirements.txt
```

---

### 10.4 OpenCV Error

If `cv2` cannot be imported, install OpenCV again:

```bash
pip install opencv-python
```

---

### 10.5 Qiskit Error

If Qiskit-related modules are missing, install the Qiskit packages again:

```bash
pip install qiskit qiskit-algorithms qiskit-optimization
```

---

### 10.6 PyTorch or CUDA Error

The code automatically uses GPU if CUDA is available. If CUDA is not available, the code runs on CPU.

CPU execution is possible, but feature extraction and QAOA optimization may take longer.
